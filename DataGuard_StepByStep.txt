🔥 MANUAL FAILOVER (Without Broker)
✅ Step 1: Standby database status check

Standby server এ login করুন:

sqlplus / as sysdba


Check করুন database role:

SELECT NAME, DATABASE_ROLE, OPEN_MODE 
FROM V$DATABASE;


Expected output:

DATABASE_ROLE = PHYSICAL STANDBY
OPEN_MODE     = MOUNTED

✅ Step 2: Managed Recovery cancel করুন
ALTER DATABASE RECOVER MANAGED STANDBY DATABASE CANCEL;


এখন recovery বন্ধ।

Verify:

SELECT PROCESS, STATUS FROM V$MANAGED_STANDBY;


MRP process থাকা উচিত না।

✅ Step 3: Last archive log applied check
SELECT THREAD#, MAX(SEQUENCE#) 
FROM V$ARCHIVED_LOG 
WHERE APPLIED='YES'
GROUP BY THREAD#;


👉 এখানে বুঝবেন শেষ কোন sequence পর্যন্ত apply হয়েছে।

⚠️ Primary পুরোপুরি dead কিনা নিশ্চিত হন
(না হলে split brain risk থাকবে)

🔥 Step 4: Standby Activate করুন (Main Failover Step)
ALTER DATABASE ACTIVATE STANDBY DATABASE;


⚠️ Important:

এটা irreversible

Standby এখন new PRIMARY হবে

✅ Step 5: Database open করুন
ALTER DATABASE OPEN;

✅ Step 6: Verify new role
SELECT NAME, DATABASE_ROLE, OPEN_MODE 
FROM V$DATABASE;


Expected:

DATABASE_ROLE = PRIMARY
OPEN_MODE     = READ WRITE

🔎 After Failover – Post Steps
1️⃣ Tempfile check
SELECT NAME FROM V$TEMPFILE;


Tempfile missing থাকলে recreate করুন।

2️⃣ Redo log status check
SELECT GROUP#, STATUS FROM V$LOG;

3️⃣ Listener & service check
lsnrctl status


Application connectivity test করুন।

🚨 Very Important Production Points
🔴 Case 1: Flashback ON ছিল?

Check করুন:

SELECT FLASHBACK_ON FROM V$DATABASE;


যদি ON থাকে → old primary পরে reinstate করা যাবে।

🔴 Case 2: Flashback OFF থাকলে

Old primary completely rebuild করতে হবে:

RMAN backup restore

Or duplicate from active database

Check mode:

SELECT PROTECTION_MODE FROM V$DATABASE;
